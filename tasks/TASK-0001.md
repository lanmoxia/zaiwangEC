# TASK-0001：验证 AI 开发闭环

> 状态：Approved  
> 创建人：User / Codex  
> Implementer：Claude  
> Reviewer：Codex  
> 关联需求：建立 Claude 开发、Codex 审查、GitHub PR、CI、中断恢复的最小闭环  
> 关联ADR：暂无  

## 1. 目标

验证一条最小 AI 协作开发链路：任务单 → Claude 实现 → PR → CI → Codex 评审 → 修复 → 复审 → 合并。

本任务不要求开发完整视频复刻业务，只用于验证开发治理流程是否可执行。

## 2. 背景

正式重启视频复刻 Agent 项目前，需要先解决 Claude 跑偏、Codex 不知道任务上下文、窗口中断无法接力、跨设备环境不一致的问题。

## 3. 范围

### 必须完成

- [ ] 初始化一个最小项目骨架，具体技术栈由用户批准后确定。
- [ ] 提供一个可运行的健康检查能力，例如 `/api/health` 或等价命令。
- [ ] 提供最小测试或验证命令。
- [ ] 提供基础 CI。
- [ ] 证明新窗口可以通过 `PROJECT_STATUS.md` 和任务单恢复上下文。

### 明确不做

- [ ] 不实现视频分析、Prompt 生成、RAG、MCP、动态 Skill。
- [ ] 不接入豆包、可灵、libTv、Qwen、Claude API。
- [ ] 不设计完整数据库模型。
- [ ] 不做 UI 设计和复杂前端页面。
- [ ] 不引入复杂 Agent 调度框架。

### 允许修改

- `CLAUDE.md`
- `AGENTS.md`
- `PROJECT_STATUS.md`
- `docs/`
- `tasks/`
- `.github/`
- 后续经用户批准的新项目骨架目录

### 禁止修改

- `release/`
- `reference_media/`
- `storyboard_work/`
- `compass-video-linker/`

- 任何发布包、压缩包、示例素材和历史实验文件
